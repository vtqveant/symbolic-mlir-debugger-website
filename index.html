Core Technology

Combining symbolic execution with MLIR compiler infrastructure

MLIR (Multi-Level IR)

MLIR is a compiler infrastructure that allows representing code at multiple levels of abstraction. Our debugger works across all these levels, providing unprecedented visibility into compiler transformations.

// MLIR code example
func.func @example(%arg0: tensor<?x?xf32>) -> tensor<?x?xf32> {
  %c0 = arith.constant 0 : index
  %result = linalg.matmul ins(%arg0, %arg0)
    outs(%arg0) -> tensor<?x?xf32>
  return %result : tensor<?x?xf32>
}

Symbolic Execution

Instead of executing code with concrete values, we use symbolic values that can represent many possible concrete values. This allows us to explore multiple execution paths in a single run.

  • Path exploration for test generation
  • Constraint solving with Z3 theorem prover
  • Automatic bug detection
  • Coverage-guided fuzzing

DAP (Debug Adapter Protocol)

Industry-standard protocol for debuggers. Enables integration with all major IDEs (VSCode, IntelliJ, etc.) and provides a consistent interface for both human developers and AI agents.

IDE Integration: VSCode, IntelliJ, Neovim, Emacs
AI Agent Integration: Structured debugging API
Protocol: JSON-RPC over WebSocket/Stdio

Agentic Workflows & Dark Factories

Designed for the future of automated software engineering

AI-Assisted Debugging

AI coding agents can use the debugger to understand complex MLIR transformations, generate test cases, and automatically fix issues. The structured debugging API provides actionable insights.

# AI agent debugging workflow
from mlir_debugger import DebuggerClient

client = DebuggerClient()
trace = client.analyze(mlir_code)
issues = trace.find_issues()
for issue in issues:
  fix = generate_fix(issue)
  apply_fix(fix)
